# School Microservices Backend (Python + FastAPI)

## 1. Project Overview

This repository contains a complete Microservices MVP for a **School Management Domain**, developed for:

- Department of Information Technology
- Faculty of Computing
- Sri Lanka Institute of Information Technology
- IT4020: Modern Topics in IT (Assignment 2)

The solution demonstrates:

- Domain decomposition into independent microservices
- Full CRUD APIs for each sub-domain
- API documentation using Swagger/OpenAPI
- API Gateway pattern to expose all services under a single entry point

## 2. Business Domain and Sub-Domains

Selected domain: **School**

Sub-domains implemented as microservices:

1. Student Service
2. Teachers Service
3. Sports Service
4. Exams Service
5. Subjects Service

Each microservice is independent, has its own process and port, and exposes REST APIs.

## 3. Technology Stack

- Language: Python 3.14+
- Framework: FastAPI
- ASGI Server: Uvicorn
- HTTP Client (Gateway proxying): HTTPX
- API Docs: Swagger UI (auto-generated by FastAPI)

## 4. High-Level Architecture

```text
Client / Browser / Postman
					|
					|  Single Port Entry (8000)
					v
		 +-------------------+
		 |    API Gateway    |
		 |   (FastAPI App)   |
		 +-------------------+
			 |      |      |      |      |
			 v      v      v      v      v
	 Student  Teachers Sports Exams Subjects
	 :8001     :8002   :8003  :8004   :8005
```

## 5. Why API Gateway Is Needed

Without a gateway, client applications must call multiple ports directly (`8001`, `8002`, `8003`, `8004`, `8005`).

With the gateway:

- Clients call a **single port** (`8000`)
- Routing is centralized by service name
- URL format is standardized
- Easier to secure, monitor, and scale later

Gateway route pattern:

```text
/api/{service-name}/{service-path}
```

Examples:

- `/api/students/students`
- `/api/teachers/teachers`
- `/api/sports/sports`
- `/api/exams/exams`
- `/api/subjects/subjects`

## 8. CRUD Endpoints by Service

All services provide the following operations:

- `POST` create record
- `GET` list all records
- `GET` get record by ID
- `PUT` update record by ID
- `DELETE` delete record by ID

### Student Service

- `POST /students`
- `GET /students`
- `GET /students/{student_id}`
- `PUT /students/{student_id}`
- `DELETE /students/{student_id}`

### Teachers Service

- `GET /teachers`
- `GET /teachers/{teacher_id}`

- `DELETE /teachers/{teacher_id}`

### Sports Service

## 9. Local Setup Instructions

### 9.1 Prerequisites

- Windows PowerShell
- Python 3.11+ (tested with Python 3.14)

### 9.2 Create virtual environment and install dependencies

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

If PowerShell blocks activation, run this once:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

## 10. Running the System

### Option A: Start everything with one command (recommended)

## 14. Troubleshooting

### Port already in use

If a service fails to start, a port may already be occupied.

Check a port:

```powershell
netstat -ano | findstr :8001
```

Kill by PID:

```powershell
taskkill /PID <PID> /F
```

### Module import errors

If you see `Import "fastapi" could not be resolved`, activate your virtual environment and reinstall dependencies:

```powershell
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### Gateway returns 404 for invalid path

Ensure URL format matches:

```text
/api/{service-name}/{service-route}
```

For example:

- Correct: `/api/students/students`
- Wrong: `/students`

## 17. Notes and Limitations

- Data is stored in-memory for MVP demonstration.
- Restarting a service resets its data.
- Authentication/authorization and persistence are intentionally out of scope for this assignment MVP.
